# Magus Cloud 技术方案与选型说明

## 1. 总体架构

系统采用前后端一体化部署架构：

- 前端使用 React 构建单页应用，编译后输出至 `dist/`
- 后端使用 Express 提供接口服务与静态资源服务
- 元数据写入 PostgreSQL
- 用户文件存放于共享存储目录
- 帮助文档由本地 Markdown 文件提供

## 2. 组件构成

### 2.1 前端

- 技术栈：React 19、TypeScript、Vite、Ant Design
- 主要职责：登录页、文件工作台、管理员后台、帮助抽屉、预览界面

### 2.2 后端

- 技术栈：Node.js、Express 5、TypeScript
- 主要职责：认证、配置管理、文件服务、分享服务、备份服务、状态服务、帮助文档服务

### 2.3 数据与存储

- PostgreSQL：存储节点状态、分享记录、备份记录、运行事件等元数据
- 共享目录：存储实际用户文件
- 配置文件：`config/magus.config.json`、`config/cloud.config.json`

## 3. 关键实现方案

### 3.1 认证方案

- 管理员登录通过系统配置与环境变量校验
- 普通用户登录通过飞书 OAuth 回调完成
- 会话通过 Cookie 保存

### 3.2 文件方案

- 文件上传使用 Multer
- 文件列表、删除、下载和目录创建通过本地文件系统实现
- 路径解析采用安全路径控制，防止目录穿越
- 存储容量与用户配额通过配置项控制

### 3.3 预览方案

- 文本、图片、PDF 直接预览
- Office 文档预览依赖容器运行时中的 LibreOffice

### 3.4 备份方案

- 站点快照将存储目录与元数据导出归档
- 用户导出按用户目录生成独立归档
- 用户导入支持 Dry Run 校验
- 备份结果记录到元数据表

## 4. 部署方案

### 4.1 单机部署

- Dockerfile 构建应用镜像
- `docker-compose.yml` 提供应用、PostgreSQL、ngrok 的单机编排

### 4.2 集群示例

- `deploy/cluster/` 提供 PostgreSQL、网关、应用节点拆分部署示例
- 多节点场景使用统一配置文件和共享存储

## 5. 选型说明

### 5.1 前端选型理由

- React 与 Ant Design 适合后台管理类界面
- TypeScript 可提高类型安全与维护性
- Vite 构建效率高

### 5.2 后端选型理由

- Node.js 与 Express 适合中小型文件服务和后台接口场景
- TypeScript 可实现前后端统一语言

### 5.3 数据选型理由

- PostgreSQL 适合结构化元数据存储
- 文件存储采用共享目录，便于直接读写与归档

## 6. 部署资源估算

以下为单套环境的基础部署建议：

| 组件 | 建议配置 |
| --- | --- |
| 应用服务 | 2 vCPU / 4 GB RAM |
| PostgreSQL | 2 vCPU / 4 GB RAM |
| 共享存储 | 按业务数据量配置，建议预留不少于 20% 空间 |
| 备份存储 | 建议独立预留归档空间 |

说明：上述为项目部署建议值，用于交付实施参考。
